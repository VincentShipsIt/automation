# ZipTax/claude

Source: https://github.com/ZipTax/claude

This is a useful example of a team-owned Claude configuration repository, but it does not include an explicit license in the repository root at the time reviewed, so it is linked instead of copied.

## What It Contains

- A `routines/` directory with one Markdown doc per live Claude Routine.
- A real routine example: `Fellow Meeting to Linear Spec`.

## Pattern Worth Copying

Use one Markdown file per routine and document:

- purpose
- connectors/MCP tools
- setup steps
- exact prompt
- API trigger instructions
- environment variables
- monitoring/debugging
- common issues

This is a good model for managing live routine prompts as reviewable repo artifacts.
